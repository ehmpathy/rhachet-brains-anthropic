# research: external product access

## remote repositories

### 1. anthropic messages api

**interface:** REST API via `@anthropic-ai/sdk`

**contract for tool use:**

#### tool definition format [1]

```json
{
  "name": "get_weather",
  "description": "Get the current weather in a given location",
  "input_schema": {
    "type": "object",
    "properties": {
      "location": { "type": "string" }
    },
    "required": ["location"]
  }
}
```

> "name: Tool identifier (regex: `^[a-zA-Z0-9_-]{1,64}$`)" [1]

> "input_schema: JSON Schema object defining expected parameters" [1]

#### tool_use content block [1]

```json
{
  "type": "tool_use",
  "id": "toolu_01A09q90qw90lq917835lq9",
  "name": "get_weather",
  "input": { "location": "San Francisco, CA" }
}
```

> "id: Unique identifier for matching with tool results" [1]

> "stop_reason: Response will have `\"stop_reason\": \"tool_use\"`" [1]

#### tool_result content block [1]

```json
{
  "type": "tool_result",
  "tool_use_id": "toolu_01A09q90qw90lq917835lq9",
  "content": "15 degrees",
  "is_error": false
}
```

> "tool_use_id: Must match the `id` from the corresponding `tool_use` block" [1]

> "is_error (optional): Set to `true` if execution failed" [1]

> "Tool result blocks must immediately follow their tool use blocks in message history" [1]

> "In user messages, `tool_result` blocks must come FIRST in the content array" [1]

#### parallel tool calls [1]

> "Claude can invoke multiple tools simultaneously" [1]

> "Return all tool results in a single user message" [1]

> "`disable_parallel_tool_use=true` with `tool_choice: {\"type\": \"auto\"}` ensures at most one tool" [1]

### 2. @anthropic-ai/sdk

**interface:** npm package

**version:** latest (check package.json)

**key types:**
- `Anthropic.Messages.Tool` — tool definition
- `Anthropic.Messages.ToolUseBlock` — tool_use content block
- `Anthropic.Messages.ToolResultBlockParam` — tool_result input

**zod utilities:** the sdk provides `zodToJsonSchema` for schema conversion (from zod to JSON Schema)

## best practices

### industry wide

1. **id correlation** — use the exact `id` from `tool_use` in `tool_use_id` of result [1]

2. **error flag** — use `is_error: true` for failed executions [1]

3. **order** — tool_result blocks must come FIRST in user message content array [1]

4. **parallel** — execute all tool calls in parallel, return all results in single message [1]

### within this repo

1. **rhachet contract** — translate `BrainPlugToolExecution` to anthropic's `tool_result` format

2. **signal map:**
   - `signal: 'success'` → `is_error: false`
   - `signal: 'error:constraint'` or `'error:malfunction'` → `is_error: true`

3. **exid correlation** — preserve anthropic's `id` as `exid` through the rhachet contract
