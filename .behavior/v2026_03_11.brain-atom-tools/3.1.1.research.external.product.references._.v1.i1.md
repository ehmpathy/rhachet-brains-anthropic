# research: external product references

## anthropic documentation

### [1] tool use overview

**url:** https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview

**key concepts:**
- tool definitions with `name`, `description`, `input_schema`
- `tool_use` content blocks in responses
- `tool_result` content blocks for continuation

### [2] tool use implementation

**url:** https://platform.claude.com/docs/en/agents-and-tools/tool-use/implement-tool-use

**key concepts:**
- tool definition format with JSON Schema
- `id` correlation between tool_use and tool_result
- `is_error` flag for failed executions
- parallel tool calls in single response
- order requirement: tool_result blocks first in content array

### [3] structured outputs

**url:** https://platform.claude.com/docs/en/build-with-claude/structured-outputs

**key concepts:**
- `strict: true` for guaranteed schema validation
- `output_config.format` for structured responses
- compatibility with tool use (both can be used together)
- model support: opus 4.6, sonnet 4.6/4.5, opus 4.5, haiku 4.5

## rhachet documentation

### [4] brain tools brief

**url:** https://github.com/ehmpathy/rhachet/blob/main/.agent/repo%3D.this/role%3Duser/briefs/brains/howto.use.brain.tools.md

**key concepts:**
- `BrainPlugToolDefinition` for tool declarations
- `BrainPlugToolInvocation` for tool calls
- `BrainPlugToolExecution` for tool results
- `genBrainPlugToolDeclaration` factory for reusable tools
- `asBrainPlugToolDict` for O(1) lookup

### [5] supplier implementation brief

**url:** https://github.com/ehmpathy/rhachet/blob/main/.agent/repo%3D.this/role%3Duser/briefs/brains/howto.for.suppliers.md

**key concepts:**
- `BrainAtom` vs `BrainRepl` contracts
- grain-based tool execution (atom outputs invocations, repl executes internally)
- episode continuation via `prompt: BrainPlugToolExecution[]`
