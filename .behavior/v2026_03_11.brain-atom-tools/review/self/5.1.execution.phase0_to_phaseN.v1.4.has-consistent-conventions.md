### 2. name patterns by layer

**domain layer** (`translateHook.ts`):
- `translate{Noun}{To|From}{Target}`
- e.g., `translateHookToClaudeCode`, `translateHookFromClaudeCode`

**infra layer** (`asJsonSchema.ts`):
- `as{OutputType}` for conversions
- e.g., `asJsonSchema`

**my code** (new infra pattern):
- `cast{Into|From}{Target}{Noun}`
- e.g., `castIntoAnthropicToolDef`, `castFromAnthropicToolUse`

### 3. is `cast` vs `translate` inconsistent?

**analysis**:
- `translate` = domain concept conversion (BrainHook <-> ClaudeCodeHook)
- `cast` = sdk type boundary cross (rhachet types <-> Anthropic SDK types)

the distinction is intentional and semantically appropriate:
- domain operations translate between business representations
- infra operations cast across sdk boundaries

**location supports the distinction**:
- `src/domain.operations/hooks/translateHook.ts` — domain
- `src/infra/cast/cast*.ts` — infrastructure

**verdict**: ✅ consistent by layer, not by repo

### 6. preposition choice: Into vs To

extant `translate` uses `To`:
- `translateHookToClaudeCode`

my `cast` uses `Into`:
- `castIntoAnthropicToolDef`

**analysis**:
- `To` pairs naturally with `translate` (translate X to Y)
- `Into` pairs naturally with `cast` (cast into mold)
- both are idiomatic english

**verdict**: ✅ acceptable — preposition matches verb semantics
