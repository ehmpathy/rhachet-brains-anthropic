# self-review: has-clear-instructions

## review scope

verified playtest instructions are followable by a foreman without prior context.

## analysis

### can the foreman follow without prior context?

**yes.** the playtest:

1. lists prerequisites explicitly (node, api key, pnpm, build)
2. provides sandbox location for file operations
3. includes complete setup commands
4. has the full playtest runner code inline (no external dependencies)
5. explains each test case with expected outcomes

the foreman does not need to understand the implementation — they only need to run the commands and verify outputs.

### are commands copy-pasteable?

**yes.** reviewed each command block:

| section | commands | copy-pasteable? |
|---------|----------|-----------------|
| setup | `pnpm install`, `npm run build`, `mkdir` | ✅ |
| setup | `cat > .temp/playtest-tools/playtest.ts` heredoc | ✅ |
| setup | `source ... && npx tsx ...` | ✅ |
| cleanup | `rm -rf .temp/playtest-tools` | ✅ |

all commands are complete — no placeholders, no absent flags.

### are expected outcomes explicit?

**yes.** each test case has specific expectations:

| test | expected outcome |
|------|-----------------|
| TEST 1 | `output: null` initially, then `{ answer: 4183 }` |
| TEST 2 | `output: { capital: 'Paris' }`, `calls: null` |
| TEST 3 | `2 tool(s) requested`, unique exids, `{ sum: 30, product: 1200 }` |

pass/fail criteria are binary:
- pass if: specific observable outcomes listed
- fail if: specific failure indicators listed

## verdict

✅ instructions are clear and followable

- foreman can follow without prior context
- commands are copy-pasteable
- expected outcomes are explicit
