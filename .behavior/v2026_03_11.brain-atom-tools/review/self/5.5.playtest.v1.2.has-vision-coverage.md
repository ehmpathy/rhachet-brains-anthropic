## analysis

### happy paths covered

the playtest's three tests cover all core flows:

1. **TEST 1** — tool invocation with continuation: verifies brain calls tool, caller executes, continuation produces output
2. **TEST 2** — direct answer: verifies brain can answer without tools when appropriate
3. **TEST 3** — parallel tools: verifies multiple invocations with unique exids

### edgey paths documented

the playtest documents instructions for manual verification of:
- error signals (modify tool to throw `BadRequestError` or `Error`)
- tools without output schema (remove schema from ask call)

these are documented rather than automated because:
- error signals require code modification to trigger
- schema-less calls are a configuration variant

### no requirements untested

cross-referenced all 15 vision behaviors. each maps to either:
- a happy path test (10 behaviors)
- the edgey paths documentation (5 behaviors)
