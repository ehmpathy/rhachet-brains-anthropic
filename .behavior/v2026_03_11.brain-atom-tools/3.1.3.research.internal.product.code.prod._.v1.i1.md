# research: internal production code patterns

the core production code for brain atoms lives in `src/domain.operations/atoms/genBrainAtom.ts`. this file implements the `BrainAtom` contract from rhachet, with anthropic as the supplier.

## pattern.4 = message array construction [EXTEND]

**location:** `genBrainAtom.ts:85-93`

```ts
const priorMessages: Anthropic.MessageParam[] =
  askInput.on?.episode?.exchanges.flatMap((exchange) => [
    { role: 'user' as const, content: exchange.input },
    { role: 'assistant' as const, content: exchange.output },
  ]) ?? [];
const messages: Anthropic.MessageParam[] = [
  ...priorMessages,
  { role: 'user', content: askInput.prompt },
];
```

**relation to wish:** must extend to handle `BrainPlugToolExecution[]` as prompt. when prompt is an array of tool executions, build `tool_result` content blocks instead of plain text. also need to emit prior `tool_use` blocks in assistant messages.

**extension strategy:**
1. detect when `askInput.prompt` is `BrainPlugToolExecution[]`
2. translate each execution to `tool_result` block with `tool_use_id`, `content`, `is_error`
3. preserve episode exchanges that contain tool_use/tool_result blocks

## pattern.5 = api call with structured outputs [EXTEND]

**location:** `genBrainAtom.ts:96-103`

```ts
const response = await anthropic.beta.messages.create({
  model: config.model,
  max_tokens: 16384,
  betas: ['structured-outputs-2025-11-13'],
  system: systemPrompt,
  messages,
  output_format: betaZodOutputFormat(askInput.schema.output),
});
```

**relation to wish:** must extend to include `tools` parameter when `plugs.tools` is provided.

**extension strategy:**
1. when `askInput.plugs?.tools` is defined, add `tools` array to api call
2. translate each `BrainPlugToolDefinition` to anthropic's tool format:
   - `name` from `slug`
   - `description` from definition
   - `input_schema` from `schema.input` via `asJsonSchema`
3. keep `output_format` for final response schema validation

## pattern.6 = response content extraction [EXTEND]

**location:** `genBrainAtom.ts:106-113`

```ts
const textBlock = response.content.find(
  (block): block is Anthropic.TextBlock => block.type === 'text',
);
const outputText = textBlock?.text ?? '{}';
const outputParsed = JSON.parse(outputText);
const output = askInput.schema.output.parse(outputParsed);
```

**relation to wish:** must extend to handle `tool_use` blocks when brain requests tool calls.

**extension strategy:**
1. check for `tool_use` blocks in `response.content`
2. if tool_use blocks present:
   - `output` is `null` (brain deferred to tools)
   - `calls.tools` contains `BrainPlugToolInvocation[]` translated from tool_use blocks
3. if no tool_use blocks:
   - parse text block as structured output (current behavior)
   - `calls` is `null`

## pattern.10 = zod to json schema conversion [REUSE]

**location:** `src/infra/schema/asJsonSchema.ts:7-21`

```ts
export const asJsonSchema = (input: {
  schema: z.ZodSchema;
}): Record<string, unknown> => {
  const { z } = require('zod');
  const jsonSchemaRaw = z.toJSONSchema(input.schema);
  const { $schema: _, ...jsonSchema } = jsonSchemaRaw as Record<string, unknown>;
  return jsonSchema;
};
```

**relation to wish:** reuse for tool input schema conversion. anthropic requires JSON Schema for `input_schema` on tool definitions.
