# research: external product claims

## model support

[FACT] structured outputs are generally available on claude opus 4.6, sonnet 4.6, sonnet 4.5, opus 4.5, and haiku 4.5 [1]

> "Structured outputs are generally available on: Claude Opus 4.6, Claude Sonnet 4.6, Claude Sonnet 4.5, Claude Opus 4.5, Claude Haiku 4.5" [1]

## tools + structured output compatibility

[FACT] you can use both json outputs and strict tool use together in the same request [1]

> "You can use both JSON outputs and strict tool use together in the same request. They solve complementary problems" [1]

[FACT] `strict: true` on tools guarantees schema validation [1]

> "When you set `strict: true` on a tool: Tool `input` strictly follows the `input_schema`" [1]

> "No type mismatches (e.g., `\"2\"` instead of `2`)" [1]

> "No missing required fields" [1]

## beta requirements

[FACT] beta headers are no longer required for structured outputs [1]

> "The `output_format` parameter has moved to `output_config.format`, and beta headers are no longer required" [1]

> "The old beta header (`structured-outputs-2025-11-13`) and `output_format` parameter will continue working for a transition period" [1]

## schema support limitations

[FACT] both json outputs and strict tool use support a subset of json schema [1]

> "Both JSON outputs and strict tool use support a subset of JSON Schema" [1]

[FACT] sdks automatically handle unsupported constraints [1]

> "The SDKs automatically handle unsupported constraints by: Removing unsupported constraints (e.g., `minimum`, `maximum`, `minLength`, `maxLength`), Adding constraint info to descriptions" [1]

## tool result format

[FACT] tool_result blocks must come first in user message content array [2]

> "In user messages, `tool_result` blocks must come FIRST in the content array" [2]

[FACT] tool_use_id must match the id from tool_use block [2]

> "tool_use_id: Must match the `id` from the corresponding `tool_use` block" [2]
