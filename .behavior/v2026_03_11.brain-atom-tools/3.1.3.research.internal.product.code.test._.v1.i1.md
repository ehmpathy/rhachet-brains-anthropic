## pattern.11 = episode continuation test [EXTEND]

**location:** `genBrainAtom.integration.test.ts:114-140`

```ts
when('[t1] ask is called with continuation via on.episode', () => {
  const resultFirst = useThen('first ask succeeds', async () =>
    brainAtomSonnet.ask({
      role: {},
      prompt: 'remember this secret code: PAPAYA99...',
      schema: { output: outputSchema },
    }),
  );

  const resultSecond = useThen('second ask succeeds', async () =>
    brainAtomSonnet.ask({
      on: { episode: resultFirst.episode },
      role: {},
      prompt: 'what was the secret code i told you to remember?',
      schema: { output: outputSchema },
    }),
  );

  then('continuation remembers context from prior exchange', () => {
    expect(resultSecond.output.content).toContain('PAPAYA99');
  });
});
```

**relation to wish:** extend for tool use continuation. test that:
1. brain requests tool → caller executes → continue with tool results
2. brain remembers tool results across episode continuation

## new test scenarios needed

based on criteria.blackbox.md:

| scenario | what to test |
|----------|--------------|
| tool invocation | brain requests tool, output null, calls.tools defined |
| tool not invoked | prompt doesn't need tools, output defined, calls null |
| tool result continuation | continue with BrainPlugToolExecution[], brain uses results |
| success signal | brain uses output from successful tool |
| error:constraint signal | brain reasons about constraint violation |
| error:malfunction signal | brain reasons about failure |
| parallel tool calls | multiple invocations, unique exids |
| tools + output schema | final output conforms to schema |
| tools without schema | final output is unstructured |
| episode + metrics | episode.exid returned, tokens tracked |
